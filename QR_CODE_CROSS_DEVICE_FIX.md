# QR Code Cross-Device Access Fix

## Problem
Previously, QR codes generated by the system contained localhost URLs (e.g., `http://localhost:5000/verify/CERT123`), which only worked on the same device. When scanning the QR code on another device, it would fail because that device cannot access localhost.

## Solution
The system now supports configurable public server addresses for QR codes. This allows QR codes to work on any device that can access your server.

## How to Configure

### Option 1: Environment Variable (Recommended for Production)

Set the `SERVER_ADDRESS` environment variable before running the app:

**On Windows (PowerShell):**
```powershell
$env:SERVER_ADDRESS = "http://yourdomain.com"
python app.py
```

**On Windows (Command Prompt):**
```cmd
set SERVER_ADDRESS=http://yourdomain.com
python app.py
```

**On Linux/Mac:**
```bash
export SERVER_ADDRESS="http://yourdomain.com"
python app.py
```

### Option 2: Create .env File

Create a `.env` file in the project root with:
```
SERVER_ADDRESS=http://yourdomain.com
```

Install python-dotenv if not already installed:
```bash
pip install python-dotenv
```

### Option 3: Edit config.py Directly

Edit `config.py` and change:
```python
SERVER_ADDRESS = 'http://yourdomain.com'  # Change from None to your address
```

## Configuration Examples

### Local Network (Within your office/home network)
```
SERVER_ADDRESS=http://192.168.1.100:5000
```
- Find your computer's IP: Run `ipconfig` (Windows) or `ifconfig` (Linux/Mac)
- Make sure port 5000 is accessible from other devices on the network
- QR codes will work on any device connected to the same network

### Public Domain
```
SERVER_ADDRESS=http://yourdomain.com
```
- Use your actual domain name
- Make sure your domain points to your server
- Ensure port 80/443 is properly forwarded if behind a router

### IP Address with Custom Port
```
SERVER_ADDRESS=http://10.0.0.5:5000
```

## Testing the Fix

1. **Set your SERVER_ADDRESS** (see above options)
2. **Restart the application**
3. **Issue a new certificate** - the QR code will now contain the public address
4. **Scan with another device** - the QR code should now work!

## How It Works

### Before (Broken):
```
QR Code contains: http://localhost:5000/verify/CERT123
Device A scans: ✓ Works (localhost is accessible)
Device B scans: ✗ Fails (localhost not accessible on Device B)
```

### After (Fixed):
```
QR Code contains: http://yourdomain.com/verify/CERT123
Device A scans: ✓ Works
Device B scans: ✓ Works
Device C scans: ✓ Works
```

## Important Notes

- **New certificates only**: Existing certificates' QR codes will still have the old localhost URLs. You'll need to reissue them to get working QR codes.
- **Network accessibility**: Make sure the address you set is actually accessible from the devices that will scan the QR codes.
- **Firewall rules**: Ensure your firewall allows incoming connections on the specified port.
- **Port forwarding**: If behind a router and using a public domain, set up port forwarding (typically 80→5000 or 443→5000).

## Troubleshooting

| Issue | Solution |
|-------|----------|
| QR codes still don't work | Restart the application after setting SERVER_ADDRESS |
| "Connection refused" error | Check if port is open and accessible from the scanning device |
| Can't access from other devices on network | Verify the IP address is correct. Run `ipconfig` on Windows or `ifconfig` on Linux/Mac |
| QR code shows localhost | Clear browser cache or reissue the certificate |

## For Development

If you're only testing locally and want it to work across devices on your network:

1. Get your local IP: `ipconfig` (Windows)
2. Set: `SERVER_ADDRESS=http://YOUR_LOCAL_IP:5000`
3. On other devices, use the same IP to scan/verify

Example:
```powershell
# On the server machine
$env:SERVER_ADDRESS = "http://192.168.1.50:5000"
python app.py
```

Then scan QR codes from any device on the 192.168.x.x network!
