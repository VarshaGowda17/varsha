# Cross-Device QR Code Fix - Implementation Guide

## Problem Solved ✓

The QR codes generated by CertChain now work on **any device** - no longer limited to localhost!

### Before (Broken)
- QR codes contained: `http://localhost:5000/verify/CERT123`
- ✗ Only worked on the same device
- ✗ Failed when scanned from other devices on the network

### After (Fixed)
- QR codes now contain: `http://192.168.1.100:5000/verify/CERT123` or your domain
- ✓ Works on any device in your network
- ✓ Works on the internet if configured with a domain

---

## Quick Setup (3 Steps)

### Step 1: Login to Admin Account
1. Open your CertChain application in a browser
2. Login with admin credentials (default: admin/admin)

### Step 2: Go to QR Code Setup
1. Click on your profile dropdown (top right)
2. Select "QR Code Setup"

### Step 3: Configure Server Address
Choose one of these options:

#### Option A: Local Network (Recommended for Testing)
```
http://192.168.1.100:5000
```
Replace `192.168.1.100` with your computer's IP address.

**To find your IP:**
- **Windows:** Open Command Prompt and type: `ipconfig`
- **Mac/Linux:** Open Terminal and type: `ifconfig` or `ip addr`

#### Option B: Public Domain (Production)
```
https://certs.yourdomain.com
```
Make sure your domain points to this server.

#### Option C: Just the IP (Auto Port Detection)
```
192.168.1.100:5000
```

## Testing the Fix

### Test 1: Same Device
1. Issue a new certificate (go to Admin → Issue Certificate)
2. The certificate details page will show the QR code
3. Scan it with your phone using any QR scanner
4. Should open the verification page

### Test 2: Different Device (Local Network)
1. Find your computer's IP (using ipconfig/ifconfig)
2. Go to Admin → QR Code Setup
3. Enter: `http://[YOUR_IP]:5000`
4. Issue a new certificate
5. From another device on the same network, scan the QR code
6. Should work perfectly!

### Test 3: Public Internet (Optional)
If you have a domain pointing to this server:
1. Go to Admin → QR Code Setup
2. Enter your domain (e.g., `https://certs.example.com`)
3. Issue a certificate
4. Scan from anywhere in the world
5. Should work!

## How It Works

### Automatic IP Detection
- The system now automatically detects your machine's local IP
- If no custom address is configured, it defaults to your local IP
- All new certificates use this address in their QR codes

### Configuration Persistence
- Settings are saved to `qr_config.json`
- Configuration persists even if the app restarts
- Can be changed anytime via the setup page

### Environment Variable Support
- Still supports `SERVER_ADDRESS` environment variable
- Useful for production deployments
- Example: `export SERVER_ADDRESS="https://certs.example.com"`

## Important Notes

### For Existing Certificates
- ⚠️ Existing certificates keep their original QR codes
- To update them, you'll need to regenerate the QR codes
- New certificates issued after setup will have the correct QR codes

### Network Requirements
- Make sure port 5000 is accessible from other devices
- If behind a firewall/router, may need to open/forward the port
- For internet access, domain must be properly configured

### HTTPS/SSL
- Recommended for production use
- QR codes should use `https://` for secure connections
- Some mobile devices may require HTTPS

## Troubleshooting

### QR Code Still Not Working on Other Devices?

**Check 1: Is the server running?**
```
The server should be accessible at the configured address from your browser
```

**Check 2: Is the port open?**
```
Windows firewall or router may be blocking port 5000
Try: ping 192.168.1.100 (use your IP)
```

**Check 3: Are devices on same network?**
```
Phone and computer should be on the same WiFi network
```

**Check 4: Did you configure the address?**
```
Go back to QR Code Setup and verify the address is set correctly
```

### Configuration Not Saving?

1. Check that `qr_config.json` is created in the project root
2. Verify write permissions in the project directory
3. Check admin permissions - setup requires admin login
4. Try environment variable instead: `export SERVER_ADDRESS="http://192.168.1.100:5000"`

## Advanced Configuration

### Using Environment Variables
```bash
# Windows PowerShell
$env:SERVER_ADDRESS = "https://certs.example.com"
python app.py

# Windows Command Prompt
set SERVER_ADDRESS=https://certs.example.com
python app.py

# Linux/Mac
export SERVER_ADDRESS="https://certs.example.com"
python app.py
```

### Docker/Containerized Setup
Set environment variable when running:
```bash
docker run -e SERVER_ADDRESS="https://certs.example.com" ... certchain:latest
```

### Custom Port
If running on a different port:
```
http://192.168.1.100:8000  # Port 8000 instead of 5000
```
